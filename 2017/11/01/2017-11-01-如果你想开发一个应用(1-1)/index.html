## 前言 ##

前言

这个系列只是想对自己的一个总结，并且逼迫自己迭代熟悉从低到高的各个知识点，如有纰漏，欢迎提出。
这个系列会以一根备忘录的应用举例，来完成一个创建web应用的系列文章。
如果你在阅读这个系列，那么默认你将掌握以下知识点：

  1. 了解JDK的安装配置
  2. 了解tomcat的安装配置
  3. 掌握基础的html和js语言
  4. 掌握java基础知识，至少能独立写出冒泡排序
  5. 默认你已经安装 IntelliJ IDEA，并激活

搭建项目结构

基本上，貌似所有程序设计都是从hello world开始的，这里也不能免俗，下面我们使用目前来说最为强大的ide—-Intellij IDEA来进行一个hello world的开发，首先，打开Idea，然后操作步骤为左上角的File菜单–>New–>Project,如图所示：

这时候，会弹出模板选择界面，选择maven项目，并且，选择从脚手架创建项目，即选择create from archetype,在弹出的原型模板中，选择maven-archetype-webapp,即基于maven的webapp，如图所示：

关于maven的内容，以后会有粗略介绍，现在只要知道这个名词即可。

然后点击Next,然后：

  • 输入groupId(com.niufennan)
  • 输入artifactId(jtodos)
  • 输入versionId(1.0.0即可)

输入完成，单击next。

groupId建议在实际项目中采取域名的倒叙，以保证唯一性，当然如果是练习则任意名称均可。

此时会出现maven的一些配置信息，默认即可，然后继续单击Next按钮。

  • 输入Project name(jtodos)
  • 输入Project Location 为项目的实际路径

点击finish，就创建了一个基于maven的webapp项目。

点击finish后，一般需等待片刻，直到idea的Maven message 控制台输出 [INFO] Maven execution finished 方能确定脚手架搭建完毕。

配置tomcat

一个web应用，要运行在web容器当中，当前比较轻量级配置简单，使用量较大的的web容器可以说激素tomcat了,我相信你已经将tomcat配置完成，下面是在idea中配置tomcat。

  • 首先点击右上角的箭头，弹出配置编辑页面

  • 点击左上角的+,然后再下拉菜单中选择 tomcat server 项，弹出菜单中选择Local项

  • 然后在tomcat的配置项依次配置

说明:

1 Tomcat项目应用，通过Configure按钮配置，通过应用的名字对应时间的tomcat项目
2 浏览器配置，即调试时启动的浏览器及url端口，端口需与tomcat配置的端口号相同，否则应用无法启动，使用默认值8080即可
3 发布调试的启动项，由于此项目为独立项目，指向本项目war包即可

配置完成后，点击应用后点击OK。

运行项目

使用最傻瓜的方式，点击右上角的绿色三角按钮，若一切配置正确，则会弹出浏览器，显示Hello world，则应用的第一步架设完成.
